### 因子算法说明
#### 因子取值逻辑

事中计算公式：现货持仓数量=期初持仓数量+买成交数量-卖成交数量
仅计算现货，默认计算证券一级分类=股票、债券、基金
由于期初持仓类型枚举较多，基于事中计算现货持仓数量，在期初消息内：仅计算持仓类型=普通/买断式回购卖出待回购/信用证券账户/余券/转融券出借证券/债券借贷借出的期初持仓数量的消息。
由于业务上限制了投资类型的枚举，目前在各消息内，仅处理对应的投资类型的消息，其它投资类型的消息暂且过滤。对于现货持仓数量，仅处理：投资类型（IVSM_TYPE）=持有至到期/AC/可供出售金融资产/FVOIC/交易性金融资产的消息。
期初持仓数量取自产品持仓表数据（日初）（DWD_AST_PD_HLDP_INFO）表的持仓数量（HLDP_VOL）
买成交数量与卖成交数量取自成交流水表（DWD_TRD_MTCH_JOUR）的成交数量（MTCH_VOL），需对买卖方向进行如下加工处理。

| 交易方向类型（TRD_DIR_TYPE） | 计算方向 | 交易方向类型（TRD_DIR_TYPE） | 计算方向 |
| --- | --- | --- | --- |
| 1->买入; 12->申购; 13->基金认购; 150->定向增发申购; 184->借入; 190->上海金买入开仓; 20->承销买入; 200->即期买入开仓; 201->即期卖出开仓; 21->分销买入; 227->发行端集合申购; 229->存放同业; 237->信用拆借拆入; 26->ETF基金申购; 3->债券买入; 32->买入开仓; 34->卖出开仓; 39->债券远期买入; 53->开基申购; 55->认购; 60->融资买入; 9->配股认购; | 买成交数量 | 11->债回售; 185->借出; 191->上海金卖出开仓; 2->卖出; 22->分销卖出; 220->利率期权卖出; 228->发行端强制赎回; 232->债回售转售; 238->信用拆借拆出; 27->ETF基金赎回; 33->卖出平仓; 35->买入平仓; 4->债券卖出; 54->开基赎回; 61->卖券还款; | 卖成交数量 |

4、事中仅计算成交流水、不计算指令、确认等消息，如盘中无成交流水的指令，则将在t+1日日初持仓体现持仓数量。

#### 因子参数说明

1、流通类型

| 参数选择 | 事中计算影响 | 事中计算影响 |
| --- | --- | --- |
| 参数选择 | 期初计算 | 成交计算 |
| 参数选择 | 产品持仓表数据（日初）（DWD_AST_PD_HLDP_INFO） | 成交流水表 （DWD_TRD_MTCH_JOUR） |
| 流通股 | 期初持仓数量取持仓表中的可流通数量： 即产品持仓表中流通标识（CIR_ID）=流通时，取持仓数量（HLDP_VOL） | 当日买入/卖出成交数量取成交表中非限售成交数量： 即成交流水表中限售标志（RES_FLAG）=非限售时，取成交数量（MTCH_VOL） |
| 非流通股 | 期初持仓数量取持仓表中的非流通数量： 即产品持仓表中流通标识（CIR_ID）=非流通时，取持仓数量（HLDP_VOL） | 当日买入/卖出成交数量取成交表中限售成交数量： 即成交流水表中限售标志（RES_FLAG）=限售时，取成交数量（MTCH_VOL） |
| 不选或全选 | 期初持仓数量取持仓表中的总数量： 即产品持仓表中，取持仓数量（HLDP_VOL） | 当日买入/卖出成交数量取成交表中总成交数量： 即成交流水表中，取成交数量（MTCH_VOL） |

#### 特殊计算逻辑

| 特殊场景 | 业务说明 | 特殊计算处理逻辑 |
| --- | --- | --- |
| ETF实物申赎（投资端） | 基金公司发行的ETF联接基金，在二级市场进行的实物申赎交易。即产品是ETF联接基金，证券组可以配置是ETF、成分券。 | 指令需支持拆分为ETF、成分券多条指令，对应业务类型： ETF基金（投资端）申购成交_基金 ETF基金（投资端）赎回成交_基金 ETF基金（投资端）申购成交_成分券 ETF基金（投资端）赎回成交_成分券 分别可支持计算ETF基金、成分券成交导致的持仓变化，其中ETF基金（投资端）申购成交_成分券，交易方向为卖出；ETF基金（投资端）赎回成交_成分券，交易方向为买入。 |
| ETF实物申赎（发行端） | 基金公司作为ETF的发行方，投资者采用实物申赎的方式向基金公司申购或赎回该ETF的指令。产品为ETF，证券组仅会配置为ETF持仓的成分券。 | 指令需支持拆分为ETF、成分券多条指令，对应业务类型： ETF基金（发行端）申购成交_基金 ETF基金（发行端）赎回成交_基金 ETF基金（发行端）申购成交_成分券 ETF基金（发行端）赎回成交_成分券 但仅需计算成分券的成交对成分券持仓的影响，其中ETF基金（发行端）申购成交_成分券，交易方向为买入；ETF基金（发行端）赎回成交_成分券，交易方向为卖出。 |
| 债券交易 | 沪深债券和银行间市场债券，若对应同一只债券，一般要进行汇总持仓计算。 通过规则配置页面的高级配置：合并计算=是，则表示配置沪深债券/银行间债券，均需要默认自动汇总多市场债券计算持仓。 合并计算=否，则单独计算单市场债券的持仓数量。 | 合并代码判断逻辑： 根据规则配置内证券内码，取证券基本信息（DWD_VAR_SCR_BASI_INFO）中	风控关联证券代码（RSKM_ASSO_SCR_INCD_ID），汇总该证券内码、风控关联证券代码的持仓。 |
| 可转债转股 | 转股指令会对应减少转股的可转债数量，增加转股数量*转换比例数量的股票； | 该业务没有成交，不会盘中计算事中，直接反映在期初持仓。 |
| 可交换债换股 | 可交换债换股也使用可转债转股业务类型；目前此类业务会拆成两笔（债转股（债券转出）、债转股（股票转入））。 | 该业务没有成交，不会盘中计算事中，直接反映在期初持仓。 |
| 存款类业务 | 无持仓数量的概念，过滤此类业务消息的成交 | 存款类的不影响现货持仓数量 |
| 担保品买入/卖出 | 担保品买入/卖出可视为普通的交易所现券买入/卖出，发生在信用账户内； | 对应业务类型：担保品买入、担保品卖出 计算在信用账户上，交易方向分别按买入、卖出去计算成交数量。 |
| 余券返还 | 余券返还业务实际为证券公司发起的非交易类业务，该业务在T+1日盘后处理。 | 该业务没有成交，不会盘中计算事中，直接反映在期初持仓。 |
| 场外开放式基金申赎 | 1、开基赎回的流程为赎回-赎回执行确认-成交确认-到账交收，成交确认时已经从在途中减少了并增加成交，交收时实际无影响 2、开基申购的流程为申购-执行确认-执行确认交收-成交确认，执行确认交收时仅资金交收，现货部分仍在途，成交确认后再累计至已成交 | 事中计算业务类型=场外开放式基金申购（成交确认）、场外开放式基金赎回（成交确认）的成交消息，分别对应的交易方向为开基申购、开基赎回，算入买成交、卖成交内。 |
| 场外开基转换 | 转换流程中，转换时仍在途，转入、转出成交确认时从在途减少，转入计入买入成交，转出计入卖出成交； | 指令对应业务类型：场外开基转换成交转出确认，但需根据转换方向，支持拆分为开基申购、开基赎回多条指令。 计算时分别按开基申购算对应指令成交的证券的买成交、按开基赎回算对应指令成交的证券的卖成交。 其中转换后的数量，由上游的转入转出数量决定，上游进行数据处理 |
| 场外开基超级转换 | 基金转换需考虑不同代销平台的超级转换比例，基金转换的指令数量一般指转出数量，因此需要使用该比例计算出转入基金的数量：即转入数量=转出数量*超级转换比例。 | 此业务目前迅策不支持，暂未计算 |
