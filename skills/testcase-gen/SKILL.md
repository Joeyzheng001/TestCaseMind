# 测试用例生成技能

## 你的任务
读取测试点 JSON 文件，对每个测试点展开为完整测试用例。

## 展开原则
根据测试点描述的场景，为每个测试点生成足够覆盖所有路径的用例，不限制数量：
- 每个独立的业务路径、分支、枚举值，都应对应一条用例
- 场景中有几条路径就生成几条，不要合并，也不要为了凑数而重复
- 目标是覆盖完整，而不是追求数量多或少

---

## 关键识别规则（必须遵守）

### 规则1：业务边界 ≠ 异常输入，必须区分
当测试点描述含 max(0,...) / "结果取0" / "最小值为0" / "不得小于" 时：
- **这是业务正常边界，预期结果是合法业务值（如0）**
- 用例标题应写：「XX超出阈值时结果取边界值」
- 预期结果应写：「返回0（业务规则：负数取0）」
- **不要写**：「返回参数错误」/ 「抛出异常」/ 「校验不通过」

错误示例：测试点「融资买入负债金额最小值为0」→ 生成「传入负数返回错误」❌
正确示例：测试点「融资买入负债金额最小值为0」→ 生成「计算结果为负数时系统自动取0」✓

### 规则2：枚举值用例必须逐一展开
当测试点中出现枚举字段（如 TRD_DIR_TYPE=60/61/69）：
- 每个枚举值必须独立生成一条用例
- 用例标题格式：「字段名=枚举值(含义)-场景描述」
- 测试步骤中明确写出枚举值的具体值

### 规则3：公式计算用例必须包含具体数值
当测试点涉及计算公式时：
- 测试数据必须给出具体数字（不能只写"适当的输入值"）
- 预期结果必须给出计算过程和最终值
- 示例：「期初=10000，买入=50000，利息=1000，卖出=20000 → 预期=41000」

### 规则4：数据来源字段必须明确
当测试点涉及从数据表取值时（如"取自持仓表HLDP_COST字段"）：
- 前置条件必须说明：「XX表中存在XX字段有效数据」
- 测试步骤必须说明：「查询XX表XX字段」
- 预期结果必须说明：「返回值与XX表XX字段一致」

## 字段说明
- case_id：TC-001 格式，全局递增不重置
- source：继承测试点的 REQ/KB/RISK
- actual_result、status：固定留空字符串，供执行时填写
- steps：用 \n 分隔每个步骤，必须有具体数值
- 输出纯 JSON 数组，不要加代码块标记
